# NEXT_STEPS — 인수인계 메모 (Windows → MacBook 이어가기)

> 이 메모는 작업 PC를 윈도우에서 맥북으로 옮기기 위한 핸드오프 문서다.
> 맥북에서 이 레포를 클론한 뒤, 새 Claude(Cowork) 세션에 이 파일을 읽히고 "이어서 진행"이라고 하면 된다.

## 0. 한 줄 요약
체대입시 상담 웹앱(`app/`, Vite+React)에 **학생 성적 기반 추천 + 종점 환산 + 작년 입결 비교** 기능을 추가하는 중. 마지막 남은 데이터는 **합격컷(입결)** 이며, 출처는 어디가(adiga) 체육 학과 입시결과.

## 1. 저장소 / 배포 현황
- 작업 레포(비공개): `https://github.com/today119/uinve` — 이 레포. 앱 소스 + 데이터 자산 + 추출 스크립트.
- 라이브 배포(공개): `https://github.com/today119/chedaepass` → **https://today119.github.io/chedaepass/** (현재 열리는 체대입시 상담 화면)
- uinve에는 GitHub Pages 자동배포용 워크플로(`.github/workflows/deploy.yml`)와 `app/vite.config.js`의 `base:'/uinve/'` 가 추가돼 있음(단, uinve는 비공개라 무료 플랜에선 공개 Pages가 안 뜸 — 공개 배포는 chedaepass 사용).

## 2. 현재 앱이 이미 가진 것 (README 참고)
- `app/src/data/admissions.json` — 2028계획안 추출: 수시 477 + 정시 247 모집단위. **반영비율·등급환산·수능영역반영·실기종목** 포함.
- `app/src/data/silgi_scoring*.json` + `silgiScore.js` — PDF 실기 채점표 정규화(22개 대학, 즉시사용 10개). 기록→점수 환산 엔진.
- 검색/카드 프로토타입 UI (전형유형·계열·실기비중·지역·종목 필터).
- 즉, **환산식/반영비율/실기 점수 엔진은 이미 있음.**

## 3. 아직 없는 것 (이번 작업의 핵심)
- **합격컷(작년 입결) 데이터** — README의 "⬜ 합격컷 데이터 별도 확보" 항목. 이게 있어야 종점-입결 비교가 완성됨.
- 학생 성적 입력 → 추천 → 종점 → 입결 비교 **UI 플로우**.

## 4. 만들 기능 (요청 사양)
1. 학생이 **수시/정시** 선택
2. **내신등급 + 모의고사 점수** 입력
3. 비슷한 수준의 학교 목록 추천
4. 추천 학교의 **실기 기록** 입력 → (기존 silgiScore 엔진으로) **종점 산출**
5. 종점을 **작년 입결과 비교**해 합격 가능성 자료 제공

## 5. 데이터 출처: 어디가(adiga) — 확인된 사실
- URL: `https://www.adiga.kr/ucp/cls/uni/classUnivView.do?menuId=PCCLSINF2000`
- **로그인 불필요.** 학과정보가 그대로 노출됨(총 7,361건).
- 좌측 "학과" 키워드 박스에 **"체육"** 입력 후 검색하면 체육계열로 필터됨.
- 표 컬럼: 학과명 / 대학명 / 지역 / 경쟁률(수시·정시) / 입학정원 / **입시결과(버튼 → 상세 입결)** / 비교.
- 데이터는 JS/AJAX로 로드됨 → 정적 fetch로는 빈 표만 옴. **렌더링 후 스크래핑하거나 내부 AJAX 엔드포인트를 찾아야 함.**

### 스크래핑 팁 (맥에서 시도할 것)
- Chrome(클로드 인 크롬)으로 페이지 열고 "체육" 필터 적용 → 네트워크 요청(read_network_requests)에서 학과목록/입시결과를 반환하는 `.do` AJAX 엔드포인트와 파라미터(학과코드/대학코드/학년도)를 찾을 것.
- 엔드포인트를 찾으면 그걸로 전국 체육 학과를 순회 수집 → JSON 정규화.
- 행마다 있는 "입시결과" 버튼이 여는 상세(수시/정시 70%컷, 충원율 등)도 같은 방식으로 엔드포인트 확인.

## 6. ⚠️ 주의 (저작권)
어디가 데이터는 KCUE(대교협) 저작물. **개인 상담 참고**는 몰라도, 공개 배포 앱(chedaepass)에 입결을 그대로 실어 재배포하면 어디가 이용약관·저작권정책에 저촉될 수 있음. 안전책: 공개·재배포 가능한 자료(대학 모집요강/전형결과 PDF, 어디가 "대입정보자료실"), 또는 체육특기자대입포털(info.kusf.or.kr) 검토. 최소한 출처 표기 + 비공개/내부용 분리 고려.

## 7. 맥북 셋업 체크리스트
1. Claude 데스크톱(Cowork) 설치 + 동일 계정 로그인.
2. `git`, `node` 확인 (`brew install node`).
3. `git clone https://github.com/today119/uinve` (비공개 → GitHub 인증 1회).
4. `cd uinve/app && npm install && npm run dev` 로 프로토타입 확인(localhost:5173).
5. Claude에 이 `NEXT_STEPS.md`를 읽히고 5번(스크래핑)부터 진행.
